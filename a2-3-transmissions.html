Передача данных

Тема A2.3 отвечает на вопрос: «Как именно данные физически и логически добираются от одного устройства до другого?». Здесь четыре больших блока: IP-адресация (как устройства называются), среды передачи (по чему идут сигналы), коммутация пакетов (как сообщение режется на куски и доставляется), и маршрутизация (как роутеры выбирают путь). Плюс — характеристики канала (пропускная способность, задержка, джиттер) и обнаружение ошибок.

2. IP-адресация (A2.3.1)

Internet Protocol (IP) — это протокол, который отвечает за маршрутизацию и доставку пакетов между сетями. В центре IP — IP-адрес: уникальное числовое имя устройства в сети.

🇬🇧 English summary — IP addressing. An IP address is a unique numeric identifier assigned to a device on an IP network so that routers know where to deliver packets. IPv4 uses 32 bits (about 4.3 billion addresses, written in dotted decimal such as 192.168.1.1) and is essentially exhausted. IPv6 uses 128 bits, written as eight hex groups, with mandatory IPsec support. Public addresses are globally unique and routable on the open internet; private ranges (RFC 1918: 10.x, 172.16–31.x, 192.168.x) are reused inside local networks. Static IPs stay fixed (servers, printers); dynamic ones are leased by a DHCP server. NAT lets many private hosts share one public address by rewriting source IP and port. A subnet mask (or CIDR suffix like /24) splits an address into network and host parts. The loopback 127.0.0.1 always refers to the local machine.
IP-адрес. Уникальный числовой идентификатор устройства в IP-сети. По нему другие устройства понимают, «куда» отправлять пакет.

🔢 2.1. IPv4 vs IPv6

Двух версий IP-адресов хватит для экзамена: IPv4 (старый, заканчивается) и IPv6 (новый, огромное пространство).

Характеристика IPv4 IPv6
Размер адреса 32 бита 128 бит
Запись десятичная с точками 192.168.1.1 (4 октета по 0–255) 8 групп по 4 шестнадцатеричные цифры через двоеточия 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Кол-во уникальных адресов ≈ 4,3 миллиарда (2³²) ≈ 340 ундециллионов (3,4 × 10³⁸)
Заголовок пакета относительно сложный, 14 полей упрощённый, 8 фиксированных полей
Поддержка IPsec (шифрование, аутентификация) опциональная обязательная
Реальный пример. IPv4 кончился ещё в 2011 году (последние блоки розданы IANA). Поэтому интернет-провайдеры массово используют NAT (см. ниже) и постепенно переходят на IPv6. В Индии и США уже более половины мобильного трафика идёт по IPv6.

🌍 2.2. Публичные и частные IP-адреса

Публичный IP (Public IP)

  • Назначается устройству, которое напрямую выходит в интернет.
  • Уникален в масштабах всего интернета — двух одинаковых одновременно быть не может.
  • Управляется IANA (Internet Assigned Numbers Authority) и региональными регистраторами.
  • Используют веб-серверы, почтовые серверы, любое устройство, к которому нужно «достучаться» извне.

Частный IP (Private IP)

  • Используется внутри частной сети (дом, офис) и не маршрутизируется в открытом интернете.
  • Определён в RFC 1918 — несколько диапазонов, разрешённых для повторного использования.
  • Чтобы выйти в интернет — нужен NAT на роутере.
  • Примеры устройств: домашний компьютер, принтер, смартфон в Wi-Fi.
Диапазон частных адресов Размер Где встречается
10.0.0.0 – 10.255.255.255 ≈ 16,7 млн (24-битный блок) Крупные корпоративные сети, ЦОД
172.16.0.0 – 172.31.255.255 ≈ 1 млн (16 блоков по 20 бит) Средние сети, VPN
192.168.0.0 – 192.168.255.255 ≈ 65 536 (256 блоков по 16 бит) Домашние и офисные Wi-Fi-сети; 192.168.1.1 — типовой адрес домашнего роутера

🔁 2.3. Loopback-адрес 127.0.0.1

127.0.0.1 — это специальный loopback-адрес: устройство отправляет пакет «само себе», не выходя в физическую сеть. Используется для тестирования сетевого стека и локальных серверов. В IPv6 такой же смысл имеет ::1 (двойное двоеточие :: сокращает подряд идущие нули). Весь диапазон 127.0.0.0 – 127.255.255.255 зарезервирован для loopback, но на практике используют 127.0.0.1.

Реальный пример. Веб-разработчик запускает локальный сервер и открывает в браузере http://127.0.0.1:3000 или http://localhost:3000 — браузер обращается к серверу, который работает на том же компьютере.

📌 2.4. Статические и динамические IP

Статический (Static)

  • Адрес постоянно закреплён за устройством, не меняется.
  • Настраивается вручную администратором.
  • Нужен серверам (веб, почта, DNS, принтер в офисе), к которым обращаются по конкретному адресу.
  • Минус: трудоёмко при тысячах устройств — нужно вести таблицу вручную.

Динамический (Dynamic)

  • Адрес назначается на время, при подключении к сети.
  • Раздаёт DHCP-сервер из пула свободных адресов.
  • При повторном подключении устройство может получить уже другой IP.
  • Плюс: экономит ограниченный пул адресов; масштабируется.

🌉 2.5. NAT — сетевая трансляция адресов

NAT (Network Address Translation) решает главную боль IPv4: адресов не хватает. NAT позволяет сотням частных адресов прятаться за одним публичным.

NAT: три частных IP «маскируются» за единственным публичным адресом роутера.

Как это работает по шагам:

  1. Устройство в частной сети отправляет пакет в интернет.
  2. NAT-устройство (обычно роутер) заменяет частный IP в заголовке на свой публичный IP. Также меняется порт-источник, чтобы потом понять, кому вернуть ответ.
  3. Запись «частный IP + порт ↔ публичный IP + порт» сохраняется в NAT-таблице.
  4. Ответ из интернета приходит на публичный IP. NAT смотрит в таблицу и пересылает пакет «правильному» устройству внутри сети.
Реальный пример. У вас дома подключены ноутбук, телефон, телевизор, два умных динамика и игровая консоль — всем шести устройствам провайдер выдал один публичный IP. Каждое получает свой частный IP 192.168.0.x, а роутер с NAT обрабатывает их совместный выход в сеть.

🧮 2.6. Подсеть и маска подсети (intro)

Маска подсети (subnet mask) — это битовый «фильтр», который делит IP-адрес на две части: сетевая часть (она же — номер подсети) и хост-часть (номер конкретного устройства). Устройства с одинаковой сетевой частью находятся «в одной подсети» и могут общаться напрямую — без роутера.

Чтобы узнать сетевую часть, делается побитовая операция AND между IP и маской.

Пример. IP 192.168.1.10 + маска 255.255.255.0 → сетевой адрес 192.168.1.0, хост .10. Все устройства 192.168.1.1 – 192.168.1.254 находятся в одной локальной сети.

В CIDR-нотации то же самое записывают как 192.168.1.0/24 — «24» означает, что первые 24 бита из 32 заняты под сетевую часть, а оставшиеся 8 — под хосты (256 адресов, из которых 254 могут быть назначены устройствам; первый — адрес сети, последний — широковещательный).

3. Среды передачи данных (A2.3.2)

Сигнал должен по чему-то передаваться. Есть три большие категории: оптоволокно, витая пара (медь) и беспроводные технологии. Для каждой среды важны 7 факторов: пропускная способность, сложность установки, цена, дальность, помехоустойчивость, затухание, надёжность и безопасность.

🇬🇧 English summary — transmission media. Three main families carry network signals. Twisted-pair copper cable (UTP or shielded STP) is cheap and easy to install, but suffers from electromagnetic interference and is usually limited to roughly 100 metres before a repeater is needed. Coaxial cable, with a central core inside braided shielding, was common in early Ethernet and is still used for cable TV and last-mile broadband. Fibre-optic sends pulses of light through glass strands, giving very high bandwidth, very long reach, low attenuation and immunity to EMI, but cabling and termination are expensive and require trained technicians. Wireless media — Wi-Fi, Bluetooth, cellular, satellite — use radio or microwave signals, offering mobility but suffering from interference, obstacles and weaker security. Exam answers should always compare at least three factors such as speed, range and reliability.

📡 3.2. Кратко по каждой среде

💡 Оптоволокно (Fibre-optic)

Как работает: данные передаются как импульсы света по тонким стеклянным или пластиковым волокнам. Используется два режима — одномодовый (на дальние дистанции) и многомодовый (короткие, внутри ЦОД).

Где применяется в жизни:

  • Магистральные каналы между ЦОД и трансатлантические подводные кабели — весь интернет «висит» на оптике.
  • FTTH (Fiber-to-the-Home) — оптика к квартире, скорости 1 Гбит/с и выше.
  • Внутри современных дата-центров между серверными стойками.

🧵 Витая пара (Twisted pair)

Как работает: пары медных проводов скручены попарно — это снижает взаимные помехи. Две разновидности:

  • UTP (Unshielded) — без экрана; дёшево, повсеместно. Категории Cat 5e, Cat 6, Cat 6a, Cat 7.
  • STP (Shielded) — с дополнительным экраном из фольги или оплётки; защищает от EMI; применяется на заводах, в больницах рядом с электромагнитным оборудованием.

Где применяется:

  • Кабельная разводка офисов (от коммутатора к каждому рабочему месту через RJ-45 розетку).
  • Соединение домашнего компьютера с роутером, когда нужна стабильность лучше Wi-Fi.

📡 Беспроводные технологии (Wireless)

Как работает: данные передаются электромагнитными волнами разных частот — радио (Wi-Fi 2,4/5/6 ГГц), инфракрасное (пульты), микроволны (Bluetooth, мобильная связь, спутник).

Где применяется:

  • Wi-Fi — школьная и домашняя сеть.
  • Bluetooth — наушники, мышь, клавиатура (короткая дистанция, низкое энергопотребление).
  • Сотовая связь (4G / 5G) — мобильный интернет.
  • Спутниковая связь — труднодоступные регионы, корабли, самолёты (Starlink).

📞 Коаксиальный кабель (Coaxial) — бонус для контекста

Это не входит в основной список Oxford-учебника, но упоминается в реальном мире: центральная медная жила в изоляции и оплётке. Раньше использовался в кабельном ТВ и Ethernet 10Base2; сейчас — в основном для кабельного интернета (DOCSIS) «последней мили».

Реальные кейсы выбора среды:
• Между ЦОД — оптоволокно (нужна сверхвысокая скорость и дальность).
• В аудитории школы — Wi-Fi (нет смысла тянуть кабель к каждому ученику).
• На заводе с тяжёлой электрикой — оптоволокно или STP (нет помех от моторов).
• Из соседнего города на дачу без оптики — спутник или сотовая связь.
• Игровой ПК, где важна задержка — витая пара (стабильнее Wi-Fi).

📝 3.3. Официальные экзаменационные вопросы

Ниже — две официальные задачи от преподавателя по разделу A2.3.2 «Среды передачи данных». Сценарий: университет, который прокладывает сеть между двумя кампусами и оборудует новую библиотеку.

Вопрос 1. Оптоволокно vs витая пара: университет соединяет два корпуса на расстоянии 3 км. 6 баллов

Контекст: университет прокладывает новые кабели между двумя корпусами кампуса, удалёнными друг от друга на 3 км. IT-отдел выбирает между витой парой (twisted pair) и оптоволокном (fibre optic).

(a) Объясни, почему затухание (attenuation) делает витую пару непригодной для прямого соединения этих двух корпусов. (2 балла)

  • Затухание (attenuation) — это потеря силы сигнала по мере его прохождения через среду на расстояние. [1]
  • Витая пара (медь) обладает высоким затуханием на больших расстояниях и обычно не может передать сигнал дальше ~100 метров без репитера, поэтому канал на 3 км без большого количества дополнительного оборудования невозможен. [1]

(b) Сравни витую пару и оптоволокно по двум критериям: восприимчивость к помехам (susceptibility to interference) и пропускная способность (bandwidth). (2 балла)

  • Помехи: витая пара использует электрические сигналы и подвержена электромагнитным помехам (EMI), тогда как оптоволокно передаёт сигнал светом и полностью невосприимчиво к EMI. [1]
  • Пропускная способность: оптоволокно даёт существенно большую пропускную способность (скорость передачи данных), чем витая пара. [1]

(c) Назови один фактор стоимости (cost) и один фактор сложности монтажа (complexity of installation), которые университет должен учесть при выборе оптоволокна. (2 балла)

  • Стоимость: оптоволоконные кабели и сопутствующее сетевое оборудование (трансиверы) значительно дороже в закупке, чем витая пара. [1]
  • Сложность монтажа: оптоволокно хрупкое, не допускает резких изгибов и требует специализированных инструментов (сварочные аппараты) и квалифицированных техников для сращивания и оконечной разделки — установка получается технически сложной. [1]
Вопрос 2. Проводная vs беспроводная передача в новой студенческой библиотеке. 6 баллов

Контекст: внутри новой студенческой библиотеки IT-команда решает, обеспечить ли сетевой доступ преимущественно беспроводным способом (Wi-Fi) или установить проводные порты витой пары на каждом столе.

Оцени (evaluate) использование беспроводной передачи по сравнению с проводной в библиотеке, рассматривая безопасность (security), надёжность (reliability) и мобильность/дальность (mobility / range). (6 баллов — нужно покрыть все три фактора для полной оценки.)

Безопасность (security) — до 2 баллов:

  • Беспроводные сигналы передаются по воздуху, поэтому их может перехватить любой человек в зоне действия (packet sniffing), если связь не зашифрована должным образом. [1]
  • Проводная (витая пара) требует физического доступа к сетевому порту или самому кабелю, чтобы перехватить данные, и поэтому изначально безопаснее от несанкционированного доступа. [1]

Надёжность (reliability) — до 2 баллов:

  • Беспроводная передача сильно подвержена помехам от физических препятствий (стены, книжные стеллажи в библиотеке) и других электронных устройств — это может вызывать обрывы сигнала и нестабильную скорость. [1]
  • Проводная передача обеспечивает выделенное, стабильное и высоконадёжное соединение, не подверженное обычным внешним помехам. [1]

Мобильность и дальность (mobility / range) — до 2 баллов:

  • Проводная передача сильно ограничивает мобильность: пользователь физически «привязан» к порту на столе, а дальность ограничена длиной кабеля. [1]
  • Беспроводная передача даёт отличную мобильность: студенты могут свободно перемещаться по библиотеке со своими устройствами, оставаясь подключёнными, что гораздо удобнее для современной библиотеки. [1]

4. Характеристики канала: пропускная способность, задержка, джиттер

На экзамене легко путают эти термины. Каждый описывает свой аспект качества связи.

🇬🇧 English summary — bandwidth, throughput, latency, jitter. These four metrics describe link quality and are often confused. Bandwidth is the theoretical maximum data rate of a channel, measured in bits per second (Mbps, Gbps) — the "pipe diameter". Throughput is the actual rate of useful data delivered; it is always less than bandwidth because of overhead, congestion and protocol limits. Latency is the time, in milliseconds, taken for a single packet to travel from sender to receiver — what gamers call ping. Jitter measures the variation in latency between consecutive packets; high jitter causes audio and video to stutter even when average latency is low. Bulk file downloads care most about bandwidth and throughput, while real-time video calls and online games depend on low latency and low jitter.
Термин Что означает Единицы Бытовая аналогия (труба + вода)
Пропускная способность (Bandwidth) Теоретический максимум данных, который канал может пропустить за единицу времени. бит/с, Мбит/с, Гбит/с Диаметр трубы.
Реальная скорость (Throughput) Фактическое количество данных, переданное за единицу времени. Всегда ≤ пропускной способности. бит/с Сколько воды реально течёт — обычно меньше, чем «влезает» в трубу.
Задержка (Latency) Время, за которое один пакет добирается от отправителя до получателя. мс Сколько секунд проходит, пока вода доберётся от насоса до крана.
Джиттер (Jitter) Вариация задержки от пакета к пакету (одни приходят быстрее, другие медленнее). мс Когда вода идёт «толчками», а не равномерно.
Почему это важно на практике:
• Для видеозвонка критичны низкая задержка и низкий джиттер — иначе «лагает» и пропадают слова.
• Для скачивания большого файла важнее всего пропускная способность — пусть начнётся медленно, но льётся быстро.
• Для онлайн-игр критична задержка (ping) — даже 50 мс между нажатием кнопки и реакцией могут стоить проигрыша.

5. Коммутация пакетов (A2.3.3)

Когда вы отправляете данные в сеть, они не идут одним куском. Они режутся на маленькие порции — пакеты — и каждый пакет отправляется независимо. Этот метод называется packet switching (коммутация пакетов) и лежит в основе всего интернета.

🇬🇧 English summary — packet switching. In packet switching, a message is divided into small units called packets. Each packet has a header (source and destination IP, sequence number, error-check code) plus a payload of data. Wrapping data with headers is called encapsulation; unwrapping at the destination is decapsulation. Packets travel independently through the network and can take different routes; the receiver reorders them using the sequence numbers and asks for any missing ones to be resent. This makes the network resilient (failed links are routed around), efficient (one channel carries many users' packets) and scalable. By contrast, circuit switching (used in the old telephone network) reserves a dedicated path for the whole conversation, giving guaranteed bandwidth but wasting capacity when idle. Within a LAN, a switch forwards frames by MAC address; between networks a router consults a routing table and forwards each packet to the next hop.
Пакет (Packet). Структурная единица данных в сети: payload (полезная нагрузка — кусок исходных данных) + header (заголовок с метаданными: адрес отправителя и получателя, номер в последовательности, контрольная сумма).

🧩 5.1. Зачем делить данные на пакеты

  • Параллельность. Разные пакеты могут идти разными путями — сеть используется эффективнее.
  • Надёжность. Если один пакет потерян или повреждён, перепосылается только он, а не всё сообщение.
  • Адаптивность. Если один канал перегружен, роутеры могут отправить следующий пакет в обход.
  • Справедливость. Канал не «забивается» одним длинным сообщением — между его пакетами проходят чужие.

🗺️ 5.2. Визуализация: один пакет — много путей

Три пакета одного сообщения идут разными маршрутами через сеть — каждый роутер выбирает путь динамически. Получатель потом собирает их по порядку.

🔄 5.3. Жизненный цикл пакета (10 шагов по учебнику)

  1. Инициация действия. Например, в Minecraft вы кликаете «поставить блок» — данные должны уйти на игровой сервер.
  2. Сегментация данных. Действие превращается в данные (координаты блока, тип, кто его поставил). Данные разрезаются на маленькие куски.
  3. Создание пакета (инкапсуляция). Каждый кусок упаковывается в пакет. Добавляется заголовок с адресами «откуда / куда».
  4. Добавление контрольной информации. В заголовок включают номер последовательности (sequence number) и код для проверки на ошибки.
  5. Независимая передача. Пакеты отправляются по сети независимо друг от друга.
  6. Маршрутизация. На каждом узле роутер решает, по какому пути отправить пакет дальше — на основании трафика и доступности. Один пакет может пойти через Нью-Йорк, другой — через Чикаго.
  7. Прибытие на сервер. Пакеты приходят, возможно не по порядку и с разной задержкой.
  8. Сборка (декапсуляция). Сервер использует sequence numbers, чтобы восстановить исходный порядок. Если каких-то пакетов не хватает — запрашивает их повторно.
  9. Обновление состояния. Игровой сервер ставит блок в игре и рассылает обновление другим игрокам.
  10. Финальный показ. Друг видит у себя в Minecraft новый блок ровно там, где вы его поставили.
Инкапсуляция / декапсуляция. Encapsulation = «обернуть» полезные данные служебной информацией (заголовки, контрольные коды). Decapsulation = «снять обёртку» на стороне получателя, чтобы достать исходные данные.

🔀 5.4. Packet switching vs Circuit switching

Packet switching (коммутация пакетов)

  • Данные режутся на пакеты, каждый идёт своим путём.
  • Канал общий: между моими пакетами идут чужие.
  • Хорошо переносит отказы — пакет находит обход.
  • Эффективно для коротких сообщений и веба.
  • Используется в интернете (IP/TCP/UDP).

Circuit switching (коммутация каналов)

  • Сначала прокладывается постоянный канал от A к B.
  • Канал занят только для этой пары, пока разговор не закончится.
  • Гарантированная пропускная способность, низкий джиттер.
  • Неэффективно: канал может простаивать, когда никто не говорит.
  • Классика — аналоговая телефонная сеть (PSTN).

🚦 5.5. Роль коммутаторов и маршрутизаторов

  • Коммутатор (Switch) работает внутри локальной сети. Пересылает пакеты на основании MAC-адресов, ведёт MAC-таблицу. Снижает «шум» в сети.
  • Маршрутизатор (Router) работает между сетями. Пересылает пакеты на основании IP-адресов. Смотрит в таблицу маршрутизации, выбирает следующий hop (узел, которому передать пакет).
Хоп (Hop). Один «прыжок» пакета от одного узла к следующему (например, от роутера R1 к R4). Число хопов до получателя — одна из метрик качества маршрута.

6. Обнаружение ошибок: parity, checksum, CRC

Любая передача — это риск, что один или несколько бит «перевернутся» из-за помех. Поэтому в каждый пакет встраивают код для проверки на ошибки. Учебник Oxford упоминает error-checking codes в составе заголовка; конкретные методы — это классика курса.

🇬🇧 English summary — error detection. Every transmission risks bit flips due to electrical noise or interference, so packets carry extra bits used to detect errors. A parity bit appends a single 1 or 0 so the total number of ones is even or odd; it catches any odd number of flipped bits but misses even-numbered errors. A checksum sums the bytes of the message into a small fixed-size value that the receiver recomputes and compares; it catches most random corruption but can miss errors that cancel out. A Cyclic Redundancy Check (CRC) treats the data as a polynomial and stores the remainder when divided by a fixed generator, catching nearly all single-bit and burst errors typical of real channels. CRC is used in Ethernet frames, Wi-Fi, USB and archive formats. These methods only detect errors; correction needs Hamming or Reed-Solomon codes, or simple retransmission as TCP does.
Метод Идея Что ловит / не ловит Где используется
Бит чётности (Parity bit) К группе бит добавляется один дополнительный бит, чтобы общее число единиц было чётным (even parity) или нечётным (odd parity). Ловит любое нечётное число ошибок (1, 3, 5 …); не ловит чётное число (2, 4 — перевернулись, но чётность сохранилась). RAM с поддержкой ECC (расширенные варианты), старые UART, простые передачи.
Контрольная сумма (Checksum) Складываются все байты данных, результат сжимается до фиксированного размера и отправляется. Получатель пересчитывает и сравнивает. Ловит большинство случайных ошибок; не ловит ошибки, которые «компенсируют» друг друга в сумме. Заголовки TCP и UDP, IP-заголовок (старая версия), контрольная сумма файлов.
CRC (Cyclic Redundancy Check) Данные «делят» на специальный полином, остаток от деления (CRC-код) добавляют к пакету. Получатель повторяет деление и проверяет. Очень мощный: ловит почти все одиночные и групповые ошибки, типичные для реальных каналов. Ethernet-кадр, Wi-Fi, ZIP/RAR-архивы, оптические диски, USB.
💡 Все эти методы только обнаруживают ошибку — для исправления нужны более сложные коды (Hamming, Reed-Solomon), либо повторная отправка пакета (это и делает TCP).

7. Кодирование и модуляция (basics)

Чтобы биты «уехали» по физическому каналу, их надо превратить в сигнал. Это две связанные задачи:

Кодирование (Encoding)

  • Преобразование битов 0/1 в форму, пригодную для канала (электрический импульс, световой импульс, радиоволна).
  • Примеры схем: NRZ (1 = высокий уровень, 0 = низкий), Manchester (переход в середине бита), 8b/10b для оптики.

Модуляция (Modulation)

  • Превращение цифрового сигнала в аналоговый (волну) для передачи по аналоговому каналу.
  • AM — амплитудная (меняется громкость), FM — частотная (меняется частота), PSK / QAM — фазовая / квадратурная (используются в Wi-Fi и 4G).
  • Демодуляция — обратное преобразование, аналог → цифра, на стороне получателя.

Эти процессы выполняет модем (см. A2.1.3 Сетевые устройства) и сетевые карты при выходе сигнала в кабель или эфир.

8. Маршрутизация: статическая vs динамическая AHL

Это тема A2.3.4 — её формально проходят только в HL, но для глубокого понимания сетей полезно знать всем.

🇬🇧 English summary — routing (AHL). A router moves packets between networks by consulting its routing table: rules that map a destination network to the next hop (the neighbouring router or output interface). With static routing, an administrator manually enters every route. This is simple, predictable and uses few resources, but does not scale and must be edited by hand whenever the topology changes — suitable for small offices or home networks. With dynamic routing, routers exchange reachability information using protocols such as RIP (uses hop count), OSPF (uses link cost, common inside an organisation) and BGP (between large internet providers). Routes are recalculated automatically when a link fails; the time for all routers to agree on the new topology is called convergence. Dynamic routing scales to thousands of routers but consumes more CPU, memory and bandwidth, and is harder to configure and secure.

📋 8.1. Что такое routing

Каждый роутер хранит routing table (таблицу маршрутизации) — это набор правил «если адрес назначения такой-то, отправь пакет в такой-то порт / на такой-то соседний роутер». Когда приходит пакет, роутер берёт IP-адрес получателя из заголовка, ищет совпадение в таблице и пересылает на следующий хоп (next hop).

🛠️ 8.2. Статическая маршрутизация

  • Администратор вручную прописывает в роутере: «для сети X next hop — Y».
  • Хранятся: адрес сети назначения, маска подсети, следующий hop или выходной интерфейс.
  • Плюсы: просто, предсказуемо, мало нагружает CPU/память; нет лишнего трафика обмена маршрутами; можно жёстко контролировать пути ради безопасности.
  • Минусы: при изменении топологии (упал линк, добавилась сеть) нужно править вручную; не масштабируется на сотни роутеров; в случае сбоя пакеты могут «стучаться в стену».
  • Где подходит: маленькие стабильные сети — небольшой офис, домашний роутер.

🤖 8.3. Динамическая маршрутизация

  • Роутеры сами обмениваются информацией о доступных сетях по специальным routing-протоколам.
  • Известные протоколы: RIP (Routing Information Protocol — старый, использует hop count), OSPF (Open Shortest Path First — внутренний протокол, считает «стоимость» канала), BGP (Border Gateway Protocol — между крупными провайдерами в интернете).
  • Метрики выбора маршрута: число хопов, пропускная способность, задержка, надёжность.
  • При изменении топологии (упал кабель) протокол пересчитывает маршруты автоматически. Время, нужное на согласованное понимание новой топологии всеми роутерами, называется convergence (сходимость).
  • Плюсы: сам адаптируется к изменениям, масштабируется на большие сети.
  • Минусы: сложнее настроить и понять; нагружает CPU/память роутера; обмен маршрутами потребляет полосу; есть риски безопасности, если протокол не защищён.

📊 8.4. Сравнительная таблица

Фактор Статическая Динамическая
Конфигурация Вручную, явные маршруты Автоматически через routing-протоколы
Сопровождение Ручные правки при любом изменении Автоматически адаптируется
Сложность Просто на маленьких сетях, тяжело при росте Сложна в настройке, но управляет ростом сама

Реальный пример. В сети одной школы (10–20 коммутаторов) обычно настраивают статическую маршрутизацию — топология стабильна. А в сети крупного оператора связи (тысячи маршрутизаторов в разных странах) работают OSPF внутри и BGP на границах — иначе администраторы не успели бы вручную поддерживать таблицы.

Что важно запомнить

  • IPv4 vs IPv6: 32 бита (≈ 4,3 млрд) против 128 бит (≈ 3,4 × 10³⁸). IPv6 — будущее, IPv4 — настоящее за счёт NAT.
  • Публичный IP — глобально уникален и маршрутизируется в интернете. Частный IP — только внутри LAN, диапазоны RFC 1918 (10.x, 172.16–31.x, 192.168.x).
  • NAT — массовая трансляция «много частных в один публичный», экономит IPv4.
  • Маска подсети делит IP на сетевую и хост-часть; в CIDR записывается как /24.
  • Среды передачи: оптоволокно — быстро, далеко, дорого, без помех; витая пара — дёшево, до 100 м, чувствительна к EMI; беспроводные — мобильно, но уязвимо к помехам и перехвату.
  • Bandwidth ≠ throughput ≠ latency ≠ jitter — все четыре про разное; типичная экзаменационная ловушка.
  • Packet switching: данные → сегментация → пакеты с заголовками → независимая маршрутизация → сборка по sequence numbers. Этим живёт весь интернет.
  • Заголовок пакета содержит IP-адреса источника и приёмника, sequence number, контрольные коды (parity / checksum / CRC).
  • Switch работает по MAC-адресам внутри LAN; router — по IP-адресам между сетями, использует routing table и выбирает next hop.
  • Static vs dynamic routing (AHL): статика — вручную, для маленьких сетей; динамика (RIP, OSPF, BGP) — автоматически, адаптивно, для больших сетей. Ключевое понятие — convergence.
  • На экзамене: вопросы «compare» и «contrast» отвечай минимум через 3 фактора; обязательно используй термины encapsulation, routing table, next hop, subnet mask, NAT.
